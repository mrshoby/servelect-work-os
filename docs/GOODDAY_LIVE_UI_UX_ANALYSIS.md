## UX principles to keep in SERVELECT WORK OS
- Work OS first: My Work, task hierarchy, views, status/workflow, resource planning and reporting must remain central.
- Dense enterprise UI: compact tables, badges, filters, right panels, evidence-driven admin screens.
- Controlled customization: task types, custom fields, custom statuses, saved views and access rules must be visible and manageable.
- Production safety: enterprise access control and custom access rules imply session-bound ACL, audit evidence and rollback drills before real writes.

## v8.1.0 design implication
Primary write/session/provider screens use clean evidence blocks, compact tables and admin-ready status badges. This improves maturity without changing Servelect identity.
